Two problems:

     correction, NCI has a systematic within-day shape even for a

     time series that looks like short-term variability and slightly 
     biases the daily median.

THE FIX:

PART A — Add IAM correction in daily.py:

Add a helper function compute_iam(aoi_deg, b0) that computes the 
ASHRAE IAM model:
    IAM = 1 - b0 · (1/cos(aoi_rad) - 1)
    IAM = clip(IAM, 0, 1)

AOI for each row can be computed from the solar position that is 
already available via orientation.py's _solar_position() function — 
do NOT rewrite solar geometry, call it directly. The AOI between 
the sun and a tilted surface is:

The pipeline currently detects curtailment in two ways:
  1. State-flag curtailment: inverter reports state codes
     513, 514, 1284, 1285 (CURTAILED_STATES in constants.py).
     These set the CURT_STATE quality flag.
  2. Statistical curtailment: power clipping detected geometrically

Both methods miss a third and common real-world mode:

  SOFT CURTAILMENT (voltage-rise throttling):

      Start with base score from layer:
        layer 1 → 1.0
        layer 2 → 0.7
        layer 3 → 0.4
      Multiply by reference method factor:
        recovery_anchored → 1.0
        p95_fallback      → 0.7
      Multiply by peer factor:
        peer_ladder_level 1 → 1.0
        peer_ladder_level 2 → 0.85
        peer_ladder_level 3 → 0.70
        peer_ladder_level 4 (no peers) → 0.55
        None (not computed) → 0.55
      Multiply by recovery count factor:
        n_recovery_events_used >= 2 → 1.0
        n_recovery_events_used == 1 → 0.85
        n_recovery_events_used == 0 → 0.65
      If peer_substituted: multiply by 0.7
        (the string's own baseline was not clean — lower trust)

    D3 — Trend quality (weight: cfg.conf_weight_trend, default 0.25)
      If slope_significant AND slope_snr >= cfg.soiling_slope_snr:
        base = min(slope_snr / 10.0, 1.0)
          (SNR of 10+ → full score; SNR of 2 → 0.2)
      Elif has_recovery_signature:
        base = 0.6
          (recovery alone is evidence even without a clean slope)
      Else:
        base = 0.2
          (no trend evidence — low confidence in soiling verdict)
      Multiply by segment quality:
        n_valid_segments / max(n_total_segments, 1)
          (if half the segments were too short to fit → penalty)

    D4 — Cross-string agreement (weight: cfg.conf_weight_agreement,
         default 0.15)
      If baseline_disagreement_flag:
        sub_score = max(0.0, 1.0 - baseline_disagreement_pp / 10.0)
          (10pp disagreement → 0.0; 0pp → 1.0; linear)
      Elif peer_ladder_level is not None and peer_ladder_level <= 2:
        sub_score = 1.0
          (peers exist and agree — no disagreement flag fired)
      Else:
        sub_score = 0.6
          (no peers to compare against — neutral)

    D5 — Sufficiency (weight: cfg.conf_weight_sufficiency, default 0.15)
      "Good"    → 1.0
      "Limited" → 0.6
      "Poor"    → 0.3
      "Skipped" → 0.0

    Final score:
      raw_score = sum(weight_i * sub_score_i for i in D1..D5)
      confidence_score = clip(raw_score, 0.0, 1.0)

    Returns a dict with:
      confidence_score: float          (0.0–1.0, the primary output)
      confidence_label: str            ("high" | "medium" | "low")
        high   → score >= cfg.conf_high_threshold   (default 0.72)
        medium → score >= cfg.conf_medium_threshold (default 0.45)
        low    → score <  cfg.conf_medium_threshold
      sub_scores: dict                 (D1–D5 individual sub-scores,
                                        for explainability)
      confidence_explainability: str   (human-readable breakdown,
                                        see format below)
    """

3. A function format_confidence_explainability(score_dict: dict,
   inputs: ConfidenceInputs) -> str:

    Produces a human-readable string like:

    "Confidence: 0.81 (high)
     D1 data_quantity=0.85 [25 days, avail=78%]
     D2 baseline=0.91 [layer=1, recovery_anchored, 2 anchors, peers=L2]
     D3 trend=0.72 [slope_snr=7.2, significant, 3/3 valid segments]
     D4 agreement=1.00 [no disagreement, peer_ladder=2]
     D5 sufficiency=1.00 [Good]"

    Keep each line under 80 characters. Use the sub_scores dict
    and the ConfidenceInputs fields to populate.
